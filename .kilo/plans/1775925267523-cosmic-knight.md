# Plano: Corrigir Navegação para Admin

## Problema
Ao clicar em "Admin" na sidebar, a página não muda - permanece no "Centro de Comando" (EmptyState ou Dashboard).

## Análise
O código em Layout.tsx parece correto:
- `navigate` atualiza `currentPage`
- Condicional `{currentPage === 'admin' && <AdminPage />}` existe na linha 136
- A sidebar passa `onNavigate` corretamente

Possíveis causas:
1. AdminPage pode estar a falhar silenciosamente (erro no fetch de dados)
2. O frontend pode estar com cache antigo
3. Pode haver um erro de JavaScript não visível

## Passos para Corrigir

### 1. Verificar Admin Page
- Admite que o componente Admin faz fetch de dados API na montagem
- Se falhar, pode não renderizar nada
- Adicionar console.log para debug

### 2. Forçar deploy limpo
- Limpar cache de build se necessário
- Fazer novo deploy

### 3. Verificar erros no browser
- Abrir DevTools Console
- Verificar se há erros JavaScript
